What is Anodyne?

Anodyne is a framework for .NET for developing fast, scalable and testable distributed systems (including web apps) with rich domain model and NoSQL database as a persistence storage.

It's highly pluggable, extremely flexible and allows to eliminate most of the usual boilerplate code.

One of the priorities of the framework is producing intention-revealing code by using industry standard design patterns and utilizing fluent interfaces (DSLs).

Some example configuration could look like this:

{% highlight csharp %} public override void OnConfigure(INodeConfigurator c) { c.UseWindsorContainer(); c.UseWindsorWcfProxyFactory(); c.UseMongoDataAccess(Configured.From.AppSettings("DatabaseServer", "DatabaseName")) .AsDomainStorage(); c.OnStartupPerform(); c.OnStartupPerform(); } {% endhighlight %}

Requirements

  • .NET 4.0
  • MVC 3 (for web application)
  • Visual Studio 2010 SP1 for working with solution
  • NuGet

Things you should know

Although Anodyne allows to write much less code you still have to understand the principles and patterns it is built upon.
